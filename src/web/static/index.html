📚 RAG System

Retrieval-Augmented Generation lets MuccheAI search your documents before answering.

How it works: documents are split into chunks, converted into vectors (embeddings) or indexed by keywords, then the most relevant chunks are injected into the LLM prompt.

Embedding LLM: best accuracy requires an embedding model (e.g. nomic-embed-text, mxbai-embed-large). If you only have a chat LLM, enable keyword fallback to use plain text search instead.

Where to put data: drop files into the RAG folder for the active database. Each database is a separate collection. ~/.muccheai/rag/<database>/

Multiple databases: create separate databases for different projects. Switch between them in the settings below.

RAG is local document search. It is separate from web search or online LLMs.
